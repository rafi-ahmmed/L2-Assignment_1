
# TypeScript-এ Type Safety: any বনাম unknown এবং Type Narrowing-এর গুরুত্ব

TypeScript-এর প্রধান লক্ষ্য হলো আমাদের কোডকে আরও নিরাপদ এবং ত্রুটিমুক্ত করা। কিন্তু টাইপ সিস্টেমের সাথে কাজ করতে গিয়ে আমরা প্রায়ই এমন কিছু ভুল করি যা টাইপ সেফটি নষ্ট করে ফেলে। আজকের ব্লগে আমরা জানব কেন `any` টাইপ এড়িয়ে চলা উচিত এবং কীভাবে `unknown` টাইপ ও Type Narrowing ব্যবহার করে কোডকে আরও শক্তিশালী করা যায়।

---

## ১. any টাইপ এবং 'Type Safety Hole'

TypeScript-এ কোনো ডাটার টাইপ হিসেবে `any` ব্যবহার করলে, TypeScript সেই ডাটার ওপর কোনো ধরণের টাইপ সেফটি প্রদান করে না। একারণেই একে **'Type Safety Hole'** বলা হয়। ফলে, ডিক্লেয়ার করা ভ্যালুতে কোনো ভুল থাকলে তা কম্পাইল টাইমে (Compile-time) ধরা পড়ে না, বরং রান টাইমে (Runtime) ধরা পড়ে। এটি কোডে অনাকাঙ্ক্ষিত বাগ বা এরর তৈরির ঝুঁকি বাড়ায়।

```typescript
let data: any = "Hello";

// 'any' ব্যবহার করার কারণে কম্পাইলার কোনো এরর দিবে না
// কিন্তু এটি রান টাইমে ক্র্যাশ করবে যদি টাইপ পরিবর্তিত হয়
data = 100;
data.toUpperCase(); // রান টাইমে এরর দিবে: data.toUpperCase is not a function

## ২. unknown টাইপ ব্যবহারের সতর্কতা

TypeScript-এ কোনো ডাটার টাইপ হিসেবে `unknown` ব্যবহার করলে, সেই ভেরিয়েবলটি নিয়ে কাজ করার সময় TypeScript অনেক বেশি সতর্ক থাকে। হয়তো অপারেশনটি ডাটা টাইপ অনুযায়ী একদম ঠিক আছে, তবুও `unknown` ব্যবহার করার ফলে কম্পাইল টাইমে এরর দেয়। এটি ডেভেলপারকে বাধ্য করে ডাটাটি ব্যবহারের আগে সঠিকভাবে যাচাই (Type checking) করে নিতে, যা রান-টাইম এরর থেকে আপনার কোডকে রক্ষা করে।

**কোড উদাহরণ:**

```typescript
let userInput: unknown = "TypeScript";

// userInput.toUpperCase(); 
// কম্পাইল টাইমে এরর দিবে: 'userInput' is of type 'unknown'.
// TypeScript এখানে ডেভেলপারকে বাধ্য করছে টাইপ চেক করতে।


## ৩. Type Narrowing: unknown টাইপকে সুনির্দিষ্ট করা

Type Narrowing-এর মানে হলো `unknown` টাইপের কোনো ভেরিয়েবলের টাইপকে সুনির্দিষ্ট বা নিশ্চিত করা। যখন আপনি কোনো ভ্যালুকে নির্দিষ্ট কোনো টাইপের সাথে তুলনা (compare) করে মিলিয়ে দেখেন (যেমন: `typeof` ব্যবহার করে), তখন TypeScript ওই ভেরিয়েবলটিকে সেই সুনির্দিষ্ট টাইপ হিসেবে ধরে নেয়। এভাবেই `unknown` টাইপের ডাটা ব্যবহার করা সত্ত্বেও, টাইপ যাচাই করার পর কোনো প্রকার এরর ছাড়াই যেকোনো অপারেশন করা সম্ভব হয়।

**কোড উদাহরণ:**

```typescript
let data: unknown = "Hello";

// Type Narrowing এর মাধ্যমে টাইপ নিশ্চিত করা
if (typeof data === "string") {
    // এখন TypeScript জানে এটি একটি string
    console.log(data.toUpperCase()); // আউটপুট: HELLO
} else if (typeof data === "number") {
    // এখন TypeScript জানে এটি একটি number
    console.log(data.toFixed(2));
}
